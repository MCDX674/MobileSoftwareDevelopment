# Lab11自适应布局实验报告

## 一、WindowSizeClass 概念简介
`WindowSizeClass` 是 Android 提供的窗口尺寸分级机制，用于根据当前应用可用窗口的大小，自动划分尺寸等级，实现多设备自适应布局。

`WindowWidthSizeClass` 三种宽度类别：
- **Compact**：竖屏手机，小屏手机（宽度 < 600dp）
- **Medium**：横屏手机、小平板（600dp ≤ 宽度 < 840dp）
- **Expanded**：平板、折叠屏、大屏设备（宽度 ≥ 840dp）

---

## 二、SportsContentType 枚举设计思路
`SportsContentType` 用于标识当前布局模式，仅包含两种类型：
- `ListOnly`：单面板，只显示列表
- `ListAndDetail`：双面板，列表 + 详情同时显示

设计原因：
1. 统一管理布局状态，代码更清晰
2. 直接对应大屏/小屏行为
3. 便于在主界面中根据类型切换布局

---

## 三、SportsListAndDetails 布局设计说明
采用 `Row` 实现左右双栏布局：
- 列表部分：`weight(1f)`
- 详情部分：`weight(2f)`

比例分配理由：
- 列表只需要展示简要信息，占 1/3 即可
- 详情需要展示大图、详细内容，占 2/3 更合理
- 使用 `weight` 可以自动适配各种大屏宽度

---

## 四、SportsAppBar 行为差异设计
- **小屏**：详情页显示返回按钮，标题切换
- **大屏**：始终显示主标题，不显示返回按钮

设计考虑：
- 小屏是页面跳转，需要返回
- 大屏是同屏双面板，不存在“返回上一页”

---

## 五、返回键处理策略
- **小屏**：详情页返回列表，列表页退出应用
- **大屏**：任何情况下按返回直接退出应用

原因：
- 小屏是多页面导航
- 大屏始终是一个主界面，无页面栈

---

## 六、实验问题与解决
1. **预览报错**  
   问题：`defaultSport` 不存在  
   解决：使用 `getSportsData().first()`

2. **PaddingValues 报错**  
   问题：`all = 0.dp` 不兼容  
   解决：改为 `PaddingValues(0.dp)`

3. **大屏布局不显示**  
   问题：未填充满屏幕  
   解决：添加 `fillMaxSize()`

4. **大屏返回不退出**  
   问题：缺少 `BackHandler`  
   解决：添加 `finish()` 退出逻辑
